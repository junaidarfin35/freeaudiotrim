# SEO — MOJO JOJO SYSTEM (MIN)

GOAL:
rank for high-intent, low-competition queries → drive traffic → retain via internal links

---

## TARGETING

* focus: long-tail, problem-based queries
* avoid broad keywords unless necessary
* match EXACT user intent

each page must answer:
“what problem is user trying to solve right now?”

---

## CONTENT STRUCTURE

USE:

H2: What is {tool / problem}
H2: How to use {tool}
H2: Why use {tool}
H2: Use cases
H2: FAQs

RULES:

* no filler content
* no generic intros
* start solving immediately
* write for humans, not bots

---

## WRITING STYLE

* direct, simple, practical
* no “AI tone”
* no buzzwords (“unlock”, “seamless”, etc.)
* short paragraphs
* clear steps when needed

---

## KEYWORD STRATEGY

* primary keyword in:

  * H1
  * first 100 words
  * 1–2 H2s
* use variations naturally
* avoid keyword stuffing

FOCUS:
intent > density

---

## INTERNAL LINKING (CRITICAL)

each page MUST include:

Use the related.JSON to populate primary and secondary tools for this page. Do not guess or randomize.

RULES:

* use contextual anchor text (not “click here”)
* link where it makes sense (not forced)

GOAL:
increase session depth
distribute authority
keep user inside ecosystem

---

## CONTENT BEHAVIOR

* answer fast
* no long intros
* no storytelling unless useful
* prioritize clarity over length

if user finds answer in <10 seconds → success

---

## SEO LOGIC

each page should:

* target ONE primary intent
* optionally capture 2–3 variations
* feel like exact answer to search query

---

## AVOID

* generic blog-style writing
* over-explaining basics
* copying competitor structure blindly
* writing for algorithms only

---

## PRIORITY

intent match > readability > structure > keywords

---

## FINAL CHECK

does this page feel like:
“this is exactly what I searched for”

if NO → rewrite
